# 数据可视化｜小组作业说明

- 课程：2026 年春季学期本科生《数据可视化》
- 文件来源：`小组作业说明-20260415.pdf`
- 文档最后修订：2026-04-15
- 作业性质：小组作业 / 课程项目
- 建议目录用途：`hw4/`

## 作业目标

应用本学期学到的可视化知识（尤其是可视化理论、`d3.js` 等），围绕一个具体主题设计并开发一个 **Web 可视化项目**。

每组需要：
- 选择一个主题（现象、事件、理论、指标等都可以）
- 使用 `d3.js` 做一个网页可视化系统
- 在课堂上进行汇报展示
- 额外提交一份小报告，说明项目完成过程

## 小组要求

- 自由组队
- 每组 `6 人`
- 需要指定 `1 名组长`
- 每组完成 `1 个 d3.js Web 可视化项目`
- 推荐将项目部署到网页上
  - 静态网页可考虑 `GitHub Pages`

## 选题说明

老师给了一批参考方向，重点偏“科学数据”，但明确允许自拟题目。

### 文档给出的参考主题

1. 诺贝尔奖得主发表记录数据集（1900—2016）
2. 探索作弊的 SCI / CSCI 期刊
3. 科学的多样性
4. 科学的颠覆性
5. 脑科学相关主题
6. 地理、历史、经济因素对科学分工的影响
7. 中国与中东欧国家论文合作情况
8. 计算机领域时间检验奖相关数据
9. 引文分类
10. 专利数据可视化
11. 科学家的流动性
12. 同行评议与性别偏见
13. Twitter 上的学者
14. 国家的科学人才流失
15. 中国高影响力数据论文榜单
16. 继续学习与在线教育
17. 中文科学文献数据集

### 选题边界

- 不局限于单一数据集
- 先定主题，再组合使用多个数据集也可以
- 可结合开放科学数据平台，如：
  - OpenAlex
  - Semantic Scholar
  - Microsoft Academic Graph

## 评分标准

本次作业在期末总评中共 `20 分`：

- 小组课堂展示：`8 分`
- 可视化系统：`12 分`

### 可视化评分重点

1. 选题有价值，目的明确，并最终达到该目的
2. 满足真实性要求
   - 标注数据来源
   - 在小报告中说明数据处理方式、可视化方式
3. 作品有故事性，内容充实，易于理解
4. 细节规范，没有明显槽点
   - 交互流畅
   - 配色合理
   - 图形选择准确
5. 小报告中明确写清团队成员分工，以及每位成员的具体贡献
